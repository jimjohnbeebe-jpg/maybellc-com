## Priority Issues

**[P1] Typography directly contradicts the brand's own anti-slop bar**
- **Why it matters:** `brand.md`'s reflex-reject list explicitly bans DM Sans, IBM Plex Sans, and IBM Plex Mono as training-data defaults — and the shipped page uses all three simultaneously (`index.html:34`, `styles.css:7-9`). For a brand whose stated differentiator is "disciplined, rational, understated" vs. generic AI defaults, shipping the exact fonts the register calls out as the giveaway undercuts the argument at the layer most visible to a trained eye.
- **Fix:** Run the font-selection procedure from `brand.md` fresh — three brand-voice words (e.g. "disciplined," "mechanical," "unhyped"), then browse a real catalog for a sans + mono pairing that isn't the Plex/DM default reflex, preserving the existing weight/size scale in `DESIGN.md`.
- **Suggested command:** `/impeccable typeset`

**[P1] The three venture cards are inconsistently resolved, not deliberately hierarchical**
- **Why it matters:** Only RepairVector (`index.html:89,96-100,116-119`) carries a flagship flag chip, a 3-item tag list, and the "documentation verified" badge. Zephyrphoto and RescueRich have none of these. Since the stated JTBD is evaluating "is this a real, competent operator," the two non-flagship ventures currently present measurably less credibility evidence, reading as unfinished rather than intentionally restrained.
- **Fix:** Either give ventures #2/#3 their own short tag sets (2-3 tags, same chip component) so the pattern reads complete-but-tiered, or strip RepairVector down to match the other two and let the flag chip alone communicate the flagship distinction.
- **Suggested command:** `/impeccable audit`

**[P2] Hero has zero next-action cue**
- **Why it matters:** The hero is headline + one paragraph, full stop — no link, no scroll cue. PRODUCT.md states the page "exists to convert cold traffic into a click," yet the first fold offers no explicit next step.
- **Fix:** Add one lightweight cue in the existing text-link vocabulary — e.g. "See the ventures →" anchored to `#ventures`, styled like `.venture__link`.
- **Suggested command:** `/impeccable clarify`

**[P2] The "documentation verified" trust badge disappears entirely below 900px**
- **Why it matters:** `.venture__badge { display: none; }` at the base rule, only becoming visible inside `@media (min-width: 900px)`. There is no alternate mobile treatment — the badge is simply gone on any phone/tablet under 900px, removing one of the page's few explicit trust signals from the majority-likely device class for cold-traffic research.
- **Fix:** Add a compact inline variant (e.g. inside `venture__meta` alongside the category label) that survives below 900px instead of vanishing outright.
- **Suggested command:** `/impeccable adapt`

**[P3] External links give no cue that they leave the tab**
- **Why it matters:** All venture links use `target="_blank" rel="noopener"` with no visual or accessible indication of a new tab. Minor surprise for first-time visitors and screen-reader users.
- **Fix:** Add a visually-hidden "(opens in new tab)" suffix, or reconsider whether `target="_blank"` is needed at all on a single-page site.
- **Suggested command:** `/impeccable harden`
